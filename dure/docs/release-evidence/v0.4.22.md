# v0.4.22 수용 증적

## 상태

`NOT_RUN`

이 기록은 v0.4.22 source metadata와 qualification 계약 변경이 실제 GPU·Docker·Ray·vLLM·NCCL
수용 검사를 통과했다는 뜻이 아님을 명시한다. 이 저장소에서 3×24GiB `PP=3` GPU harness,
network/NCCL 측정, distributed model load·최소 추론, Controller apply·verify를 실행한 결과는 아직
이 버전의 증적으로 제출되지 않았다.

## 대상과 전제 조건

- node UUID와 선택 GPU UUID·index: 실제 수용 노드를 선택하지 않음
- 사설 network·Ray 포트 정책 확인: 실제 환경에서 실행하지 않음
- exact cache `READY`와 image preparation evidence: 실제 환경에서 실행하지 않음
- 누락된 전제 조건: 승인된 3×24GiB GPU 노드, 고정 model/runtime digest, exact inventory와
  network/NCCL 측정, 운영자의 명시적 opt-in

## 판정과 후속 조치

- release/generation 판정: 실제 GPU/NCCL 수용 성공 증거 없음. 이 파일은 release 승인, 배포 승인,
  `VALIDATED` stage 승격 또는 장기 안정성 증거로 사용할 수 없음.
- 필요한 명시적 조치: [릴리스 수용 검증](../release-validation.md)의 사전 조건을 충족한 격리 환경에서
  opt-in harness와 Controller 검증을 수행하고, 그 실행의 source commit·package/build·model/runtime
  digest·node/GPU UUID·inventory identity·구조화된 결과를 새 evidence로 기록.
- 재사용 금지 조건: 모델·revision·manifest·runtime digest·profile·node/GPU UUID·network interface·
  inventory identity 중 하나라도 달라지면 이 `NOT_RUN` 기록은 새로운 실행에 대한 근거가 될 수 없음.
